# Release readiness checklist

## Pre-release

- Validate scope, owners, and rollback plan
- Confirm release notes and customer-facing changes are documented
- Verify monitoring and alert updates are ready
- Confirm security, compliance, and dependency checks are complete

## Release window

- Assign incident commander and communication owner
- Validate dashboards and deployment health signals
- Record release timestamps and checkpoints

## Post-release

- Confirm customer validation and smoke checks
- Log follow-up defects and documentation gaps
- Close the release summary with lessons learned
